# Radar-2D
ENG:
The 2D radar project was developed to create a simple system capable of detecting objects in the surrounding environment and visualizing them graphically. The system allows scanning of the surrounding space within a 360° range and presents measurement results on a computer in the form of a plot resembling the operation of a classic radar. Solutions of this type are commonly used in robotics, navigation systems, and simple environment-mapping applications.

The system is based on the STM32F303K8 microcontroller, which is responsible for controlling all system components and processing the measurement data. Two ultrasonic distance sensors, HC-SR04 and HY-SRF05, were used to detect objects. These sensors operate by sending an ultrasonic pulse that reflects from an obstacle and returns to the sensor. Based on the time it takes for the signal to return, the distance to the detected object can be calculated. The sensors used in the project have a measurement range from approximately 2 cm to 400 cm; however, for better readability of the visualization it was limited to about 40 cm.

To enable scanning of the environment, the sensors were mounted on an SG90 servo motor that provides rotation within a range from 0° to 180°. The sensors were mounted in opposite directions, which allows the system to achieve a full 360° scanning range. The servo motor is controlled by a PWM signal generated by the microcontroller. The PWM signal period is approximately 20 ms, while the control pulse width changes depending on the desired servo angle. For an angle of 0° the pulse width is about 0.5 ms, while for 180° it is about 2 ms. During operation, the microcontroller changes the servo position every approximately 240 ms by 5°, which allows gradual scanning of the entire space around the device.

The microcontroller is also responsible for handling the ultrasonic sensors. First, a short trigger pulse is generated to start the measurement, after which the system measures the duration of the returning echo signal. The microcontroller timers are used for this purpose, allowing precise time measurement. Based on the measured time, the distance to the object is calculated. Measurement data from both sensors are then prepared for transmission to the computer.

Communication between the microcontroller and the computer is performed via the UART interface. Data are transmitted at a speed of 115200 bits per second. The transmitted information contains three values: the current angle of the servo motor and two distances measured by the ultrasonic sensors. These values are sent continuously, which allows real-time processing by the computer application.

To visualize the measurements, a graphical interface running on a computer was developed. One version of the interface was written in Python using the Visual Studio Code environment. The program uses the PySerial library for serial communication, Matplotlib for visualization of the data in the form of a radar-style plot, and NumPy for converting polar coordinates into Cartesian coordinates. The program receives data sent by the microcontroller, processes them, and displays them on the screen as a 2D radar plot. Points on the plot are drawn only when the detected object is within a distance not greater than 40 cm. When an object is detected, the program also displays information about its position.

An alternative graphical interface was also created in the LabVIEW environment. The program connects to the device using a serial port through the VISA module. The received data are then converted from text to numerical values, and the angle given in degrees is converted into radians. Based on this data, a polar plot representing the current measurement results is generated. The interface also allows modification of the display parameters of the plot, such as scale, color, or value range, and it includes a “stop” button that allows the user to terminate the program.

The completed project demonstrates that it is possible to build a functioning object detection and visualization system in real time using relatively simple and inexpensive components. The STM32 microcontroller provides sufficient computational power and appropriate communication interfaces to support such a system, while ultrasonic sensors offer a simple and effective method for measuring distance. In the future, the project could be further developed by implementing more advanced data processing methods, measurement noise filtering algorithms, or improved visualization features, such as a three-dimensional model of the surrounding environment.

PL:
Projekt radaru 2D został opracowany w celu stworzenia prostego systemu umożliwiającego wykrywanie obiektów w otoczeniu oraz ich wizualizację w formie graficznej. Układ pozwala na skanowanie przestrzeni w zakresie 360° i przedstawienie wyników pomiarów na komputerze w postaci wykresu przypominającego działanie klasycznego radaru. Tego typu rozwiązania znajdują zastosowanie między innymi w robotyce, systemach nawigacyjnych oraz w prostych systemach mapowania otoczenia.

Podstawą systemu jest mikrokontroler STM32F303K8, który odpowiada za sterowanie wszystkimi elementami układu oraz przetwarzanie danych pomiarowych. Do wykrywania obiektów wykorzystano dwa ultradźwiękowe czujniki odległości HC-SR04 oraz HY-SRF05. Czujniki te działają na zasadzie wysyłania impulsu ultradźwiękowego, który po odbiciu od przeszkody wraca do sensora. Na podstawie czasu powrotu sygnału możliwe jest obliczenie odległości od wykrytego obiektu. W projekcie zakres pomiarowy czujników wynosi od około 2 cm do 400 cm, jednak w celu zwiększenia czytelności wizualizacji został ograniczony do około 40 cm.

Aby umożliwić skanowanie otoczenia, czujniki zostały zamontowane na serwomechanizmie SG90, który zapewnia obrót w zakresie od 0° do 180°. Czujniki zostały zamocowane przeciwnie do siebie, co pozwala na uzyskanie pełnego zakresu skanowania wynoszącego 360°. Serwomechanizm sterowany jest sygnałem PWM generowanym przez mikrokontroler. Okres sygnału PWM wynosi około 20 ms, natomiast szerokość impulsu sterującego zmienia się w zależności od kąta ustawienia serwa. Dla kąta 0° impuls wynosi około 0,5 ms, natomiast dla 180° około 2 ms. W trakcie pracy mikrokontroler zmienia kąt ustawienia serwomechanizmu co około 240 ms o 5°, dzięki czemu możliwe jest stopniowe skanowanie całej przestrzeni wokół urządzenia.

Mikrokontroler odpowiada również za obsługę czujników ultradźwiękowych. W pierwszej kolejności generowany jest krótki impuls wyzwalający pomiar, po czym układ mierzy czas trwania sygnału powrotnego. Do realizacji tego zadania wykorzystywane są timery mikrokontrolera, które umożliwiają dokładny pomiar czasu. Na podstawie zmierzonego czasu obliczana jest odległość obiektu od czujnika. Dane pomiarowe z obu czujników są następnie przygotowywane do przesłania do komputera.

Komunikacja pomiędzy mikrokontrolerem a komputerem odbywa się za pomocą interfejsu UART. Dane przesyłane są z prędkością 115200 bitów na sekundę. Format wysyłanych informacji zawiera trzy wartości: aktualny kąt ustawienia serwomechanizmu oraz dwie odległości zmierzone przez czujniki ultradźwiękowe. Dane te przesyłane są w sposób ciągły, co umożliwia ich bieżące przetwarzanie przez aplikację komputerową.

Do wizualizacji pomiarów przygotowano interfejs graficzny działający na komputerze. Jedna z wersji interfejsu została napisana w języku Python przy użyciu środowiska Visual Studio Code. W programie wykorzystano biblioteki PySerial do obsługi komunikacji szeregowej, Matplotlib do wizualizacji danych w formie wykresu oraz NumPy do przeliczania współrzędnych biegunowych na kartezjańskie. Program odbiera dane przesyłane przez mikrokontroler, przetwarza je oraz wyświetla na ekranie w postaci wykresu przypominającego radar 2D. Punkty na wykresie rysowane są tylko wtedy, gdy wykryty obiekt znajduje się w odległości nie większej niż 40 cm. W przypadku wykrycia obiektu program wyświetla również informacje o jego położeniu.

Alternatywnie przygotowano również wersję interfejsu graficznego w środowisku LabVIEW. Program łączy się z urządzeniem za pomocą portu szeregowego przy użyciu modułu VISA. Odbierane dane są następnie konwertowane z tekstu na wartości liczbowe, a kąt podany w stopniach przeliczany jest na radiany. Na podstawie otrzymanych danych tworzony jest wykres biegunowy przedstawiający aktualne wyniki pomiarów. Interfejs umożliwia także zmianę parametrów wyświetlania wykresu, takich jak skala, kolor czy zakres wartości, a także zatrzymanie działania programu za pomocą przycisku „stop”.

Wykonany projekt pokazuje, że przy użyciu stosunkowo prostych i niedrogich komponentów możliwe jest stworzenie działającego systemu wykrywania obiektów oraz ich wizualizacji w czasie rzeczywistym. Mikrokontroler STM32 zapewnia wystarczającą moc obliczeniową oraz odpowiednie interfejsy komunikacyjne do obsługi takiego systemu, natomiast czujniki ultradźwiękowe stanowią prosty i skuteczny sposób pomiaru odległości. Projekt może być w przyszłości rozwijany poprzez dodanie bardziej zaawansowanych metod przetwarzania danych, filtrowania szumów pomiarowych lub rozbudowę wizualizacji, na przykład o trójwymiarowy model otoczenia.
